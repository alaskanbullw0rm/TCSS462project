# Project Overview
This project evaluates the quality and performance of Java code generated by three Large Language Models (LLMs): ChatGPT, Google Gemini 3.5, and Microsoft Copilot. The study focuses on a serverless image processing pipeline deployed on AWS Lambda, assessing metrics like code maintainability, runtime, and throughput.

# The Pipeline
The pipeline consists of three sequential functions triggered via Amazon S3:
Rotate: Rotates the ionput image 90 degrees clockwise. 
Resize: Resizes the image to 128 x 128 pixels using bilinear interpolation.
Greyscale: Converts the image to grayscale using the standard luminosity formula.

# Technical Stack
Language: Java
Cloud Provider: AWS (Lambda, S3, CloudWatch, IAM)
Testing Tools: FaaSRunner
Dataset: STL-10 Image Recognition Dataset and custom images ranging from 5KB to 5.9MB

# Key Research Questions
1. How does LLM-generated code vary in quality and maintainability
2. What are the performance differences (latency, runtime, throughput) between pipelines generated by different LLMs?

# Results
# LLM                  Source Lines of Code (SLOC)    Observations
Google Gemini 3.5    83–98                          Most concise; easiest to decipher.
ChatGPT              214–239                        Complex nested try-catch structures; hardest to maintain.
Microsoft Copilot    199–217                        Inconsistent error handling with multiple catch calls.

# Execution Metrics
Highest Efficiency: Microsoft Copilot achieved the highest throughput for medium and large images.
Fastest Runtime: Google Gemini 3.5 consistently produced the lowest server-side runtime across all image sizes.
Best User Experience: ChatGPT provided the lowest latency and round-trip time (RTT).

# Conclusion
While all LLM-generated code compiled successfully, significant manual modification was required to integrate specialized frameworks like SAAF. The study concludes that while LLMs are effective for boilerplate, performance-critical cloud workloads still require human oversight to manage complexity and maintainability.
